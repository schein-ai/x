How we ship AI agents that hold up in production.

github.com/xrd-technologies/agent-eval
OpenMIT licence
4Engagements
2026Released
Summary

Many AI agent demos work in the demo and break in production. The reasons are familiar: prompts that drift under unexpected input, evaluation runs that pass for the wrong reasons, tool integrations that look connected but aren't observed. This write-up documents the framework we use to ship AI agents that survive contact with real business operations.

We cover prompt scaffolding, tool-call contracts, evaluation harnesses with negative-example coverage, and the operational rituals (logging, replay, human-in-loop overrides) that we run on every shipped agent. Examples drawn from four production engagements in 2025–2026. The harness itself is open-source under MIT at github.com/xrd-technologies/agent-eval. Read the full write-up →

How we work

Try first, then share.

Every R&D track cycles through the same four-phase loop. Plain names because that's what the calendar looks like. We don't write up something we haven't already used internally on a real engagement.

01

Open the question

A short note from the team that owns the track. We name the question, the kind of answer worth chasing, and the engagement we'll test it on.

Track ownerWorking brief
02

Build the prototype

Working software, prompt libraries, evaluation harnesses, training datasets — whatever the question requires. Always paired with a real engagement within weeks, not quarters.

PythonTypeScriptOpen weightsCloud
03

Run & measure

Deploy on a live engagement, observe operationally, A/B against the baseline. Every result has the failure modes alongside it — both go into the writeup.

Production logsEval harnessHuman-in-loop
04

Publish & share

Open write-up first, then the harness or library at MIT licence. Methods we write up flow back into the four businesses within a quarter — or we archive them honestly.
