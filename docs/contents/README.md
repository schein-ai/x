# 実コンテンツ

> 英語版（および各言語版）ページのダミーを差し替えるための、**実データの単一ソース**。`company.md` の §5（4 事業）構造に沿って、各事業の概要・提供メニュー・実績案件を集約する。

## 構成

```
docs/contents/
├── README.md                           ← このファイル（インデックス）
├── projects-portfolio.md               ← 全案件フラット一覧 + サイト割付
├── 01-app-development.md               ← ① AI 駆動アプリ・システム開発（8 件）
├── 02-visual-production.md             ← ② AI ビジュアル制作（5 件）
├── 03-adoption-consulting.md           ← ③ AI 導入支援・コンサル（5 件）
└── 04-talent.md                        ← ④ AI ネイティブ人材育成・派遣（4 件）
```

**サイトのカードを埋める時は `projects-portfolio.md` から引く** — どの P-ID をどのスロットに割り付けるかの提案つき。事業ごとの詳細は 4 つの事業ファイルに。

## 各ファイルのスキーマ

各事業ファイルは以下の構成:

1. **事業概要** — `company.md §5` のコピー（簡略化）
2. **提供メニュー** — typical services
3. **強み** — 差別化ポイント（`company.md` ベース）
4. **実績案件** — 1 案件 1 ブロック、構造化メタデータ付き
5. **TODO** — 確定が必要な事項

各実績案件のメタデータ:

| フィールド | 内容 |
|---|---|
| **Project ID** | `P-XXX`（採番） |
| **タイトル** | 案件名（日本語） |
| **タイトル（EN）** | 英語版ページ用 |
| **業種** | `Manufacturing · JP` 形式（公開可の粒度） |
| **ステータス** | `Delivered` / `Built` / `Running` / `PoC` / `Concept` |
| **時期** | 着手・完了年 |
| **概要** | 1〜3 文 |
| **規模／成果** | 数値が出せれば |
| **使用技術** | 主要スタック |
| **公開可否** | 業種のみ／社名公開可／非公開 |

## 凡例

| マーク | 意味 |
|---|---|
| ✏ **TODO** | 確定情報が必要 |
| 🔒 **NDA** | 公開不可（社名・詳細は伏せる） |
| 🌐 **公開可** | クライアント許諾済み、サイトに掲載可 |

## 公開ポリシー（全案件共通）

- **業種・時期・期間・規模・ステータス**: 公開可
- **社名**: 全件 **非公開**（業種粒度のみ）
