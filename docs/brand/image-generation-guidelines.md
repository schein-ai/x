# xR&D Technologies Image Generation Guidelines

> Brand direction for generating website imagery across the English site before the Next.js and multilingual rollout.
> Source references: `docs/brand/*`, `docs/company.md`, `docs/content-en.md`, and the Japanese copy model in `docs/copy/*`.

## 1. Brand Essence

xR&D Technologies is an AI-native company from Da Nang, Vietnam. It does not simply advise companies to use AI; it builds with AI, creates with AI, teaches AI-native workflows, and grows AI-native talent through its own pipeline.

The imagery should make this position feel calm, credible, and distinctive:

- AI-native, but not sci-fi.
- Technical, but not cold.
- Coastal and spatial, but not an architecture firm unless the page topic needs it.
- Human-centered, but not stock-photo corporate.
- Premium B2B, but not luxury for its own sake.
- Built in Vietnam, delivered to Japan.

The visual system should suggest that xR&D turns invisible AI work into visible, usable structure.

## 2. Core Visual Principles

### 2.1 Structured Intelligence

Use spatial structure as the main metaphor for AI. Prefer architectural curves, precise grids, network nodes, clean interface layers, and modular systems over glowing brains, robot figures, server rooms, or chaotic dashboards.

The image should feel designed, not decorated. AI should appear as a disciplined layer that clarifies the scene.

### 2.2 Coastal Light

The brand references use Da Nang as atmosphere: water, morning or late-afternoon light, pale sky, glass, concrete, vegetation, and a sense of air. Even when the subject is software, consulting, or talent, the image can carry a quiet coastal brightness.

Use Da Nang as a strategic base, not as a low-cost outsourcing backdrop.

### 2.3 Human-Centric Systems

Show people as makers, learners, collaborators, and operators. They should be present enough to humanize the work, but not posed like generic business stock photography.

When no real team photography is available, avoid identifiable faces. Use side profiles, silhouettes, hands at work, back views, small groups in conversation, or environmental portraits where identity is not the focus.

### 2.4 Light AI, Not Loud AI

AI overlays should be elegant and restrained. The best references show thin luminous lines, semi-transparent network grids, contour marks, and small nodes integrated with architecture or UI surfaces.

Avoid the excessive "blue command center" look: huge holographic dashboards, dense unreadable interface walls, neon storms, cyberpunk darkness, and every surface covered in data.

### 2.5 Real Enough, But Not Falsely Specific

Most website images can be generated as brand visuals, not documentary proof. Do not invent specific customer logos, real client names, real addresses, public company dashboards, or measurable outcomes inside images.

If a visual needs to imply a client project, keep all screens abstract, fictional, and unreadable.

## 3. Visual DNA From the Brand References

The `docs/brand` images establish these reusable ingredients:

- Minimal white or pale backgrounds with deep navy typography.
- A refined coastal/futuristic architectural setting.
- Soft water, skyline, vegetation, glass, concrete, and warm sunlight.
- Luminous blue AI line overlays that wrap or trace physical forms.
- Network-node diagrams and geometric grid fragments.
- Clean product mockups on laptop, tablet, and mobile devices.
- Small UI cards with blue accents, generous whitespace, and light gray panels.
- A composed, editorial layout rather than a saturated advertising collage.

Translate this DNA into the current AI-native company positioning. The site is no longer presenting itself as an "architectural AI startup" only. Architectural language should be used as a visual metaphor for structure, clarity, and making systems tangible.

## 4. Color System For Generated Images

Use the brand palette as the primary color reference:

| Role | Color | Hex | Usage |
|---|---:|---:|---|
| Deep brand navy | Ocean Blue | `#0B1D3A` | Primary text overlays, shadows, deep background accents |
| Primary blue | Luminous Blue | `#1E68FF` | AI lines, buttons, UI highlights, active nodes |
| Electric cyan | AI Cyan | `#00D6FF` | Sparse highlights, data edges, light traces |
| Soft gray | Concrete Gray | `#D9DBE1` | UI panels, devices, concrete, neutral background |
| Warm neutral | Coastal Sand | `#F2E6D3` | Morning light, stone, human warmth |
| Accent violet | Luminous Violet | `#8F7BFF` | Very limited secondary glow only |

Color balance:

- Default page imagery should be light, airy, and coastal.
- Dark navy images are acceptable for contrast-heavy heroes, but they should not become the whole brand.
- Blue and cyan should behave like signal, not floodlight.
- Violet is optional and should be rare.
- Avoid one-note blue/purple gradients, heavy orange sunsets, beige-only palettes, and generic black cyber dashboards.

## 5. Materials And Textures

Preferred materials:

- Glass, soft reflections, clean device screens.
- White concrete, pale stone, brushed aluminum.
- Water, sky haze, coastal atmosphere.
- Greenery used sparingly and architecturally.
- Matte work surfaces, notebooks, sketches, laptops, tablets.
- Thin translucent interface layers.

Avoid:

- Server racks as the main image.
- Dark gamer-style RGB lighting.
- Generic office cubicles.
- Overly glossy futuristic interiors.
- Fake "metaverse" avatars.
- Robot hands, robot heads, androids, brain icons.
- Dense floating panels with tiny fake text.

## 6. AI Graphic Motifs

Use these motifs across generated images:

- AI line overlays: thin, precise, curved or contour-following paths.
- Network nodes: small blue points connected by fine lines.
- Architectural contours: topographic or facade-like linework.
- Half-wireframe forms: a real object partly traced by a technical overlay.
- Grid fragments: subtle, sparse, and aligned with the composition.
- UI cards: clean white panels with blue accents, no readable confidential data.

Overlay density:

- Background hero image: medium density is allowed if text remains readable.
- Case study image: low to medium density.
- Editorial or insight image: low density.
- Product mockup image: low density, with the product screen as the main focus.
- People or talent image: very low density. Do not cover faces or bodies with effects.

## 7. Typography Inside Images

As a rule, do not generate text inside images. Website typography should be handled by HTML/CSS.

If mock UI text is required, use short non-sensitive labels, keep them mostly abstract, and accept that the image should not depend on the text being readable.

Do not generate:

- Client names.
- Real company logos.
- Legal or financial figures.
- Exact performance metrics unless already approved in source content.
- Japanese or Vietnamese text inside images unless specifically required and verified.

## 8. People Direction

When people appear:

- Show small teams, focused collaboration, teaching, reviewing work, or building.
- Favor natural posture: standing beside a table, reviewing screens, sketching, teaching, testing, presenting.
- Clothing should be modern, simple, and realistic for a Da Nang technology studio.
- Avoid exaggerated executive boardroom scenes.
- Avoid handshake shots, staged smiles, headset call-center imagery, and generic "diverse team around laptop" stock composition.
- Avoid identifiable faces unless real approved team photos are provided.

For talent and training imagery, the mood should be "grown through practice," not "recruited from a marketplace."

## 9. Place Direction

Da Nang should appear through atmosphere and geography:

- Sea, river, soft mountains, skyline, bridges, tropical light, and open air.
- Studio interiors with coastal daylight.
- University or training references can be suggested through classrooms, workshops, desks, and mentor-student scenes, but should not claim a specific partner university unless confirmed.

Do not portray Vietnam as cheap labor, crowded outsourcing rooms, or generic Southeast Asia imagery. The place should feel strategic, capable, and proud.

## 10. Page-Level Image Direction

### Home

Goal: introduce xR&D as an AI-native company from Vietnam.

Use bright, spatial, coastal imagery with enough negative space for large typography. The image may combine a modern studio, Da Nang atmosphere, and light AI linework. It should feel like the company is building something real, not pitching an abstract technology trend.

Avoid dark dashboards as the first impression.

### About

Goal: explain origin, practice, people, Da Nang, and the three commitments.

Use documentary-style brand visuals: studio fragments, coastal city context, founder/partner silhouettes, training moments, whiteboards, notebooks, worktables, and environmental details. Mosaic images can be more varied, but they should still share light, structure, and restrained blue signal.

For Tomoki / Nghia / Studio sections, use real photography when available. Until then, use non-identifiable environmental portraits or symbolic workspace scenes.

### Solutions

Goal: show the four businesses as one coherent system.

Use clean conceptual images with service-specific cues:

- AI-driven app/system development: product screens, code/workflow structure, prototype review, app interfaces on devices.
- AI visual production: visual boards, generated campaign variations, image/video production pipeline, clean creative workspace.
- AI adoption consulting: workshop table, process maps, governance/roadmap boards, decision frameworks.
- AI-native talent: training, mentorship, intern-to-team pipeline, Japanese/Vietnam collaboration, classroom-to-studio path.

Keep the service imagery modular and consistent. It should feel like four parts of the same company, not four unrelated stock categories.

### Solution Detail Pages

Goal: make each service tangible.

Use product-like or process-like images rather than abstract hero art. A detail page should show what the work produces: a dashboard, prototype, visual pipeline, agent flow, workshop output, training cohort, or delivery environment.

Images should be practical and useful, with one clear subject.

### Projects

Goal: make anonymous case studies credible without revealing clients.

Every project image should communicate:

- The business category.
- The operational context.
- The type of artifact delivered.
- The scale or rhythm of the work.

Do not generate fake client brands. Use abstracted environments, anonymized UI, industry cues, and xR&D's visual language.

For cross-business projects, show the connection between two work streams with a calm structural metaphor, not an overloaded command center.

### Project Detail Pages

Goal: support the narrative of one engagement.

Use a small set of related images:

- Hero: one strong, elegant image that establishes the engagement context.
- Gallery image 1: the input problem or operational environment.
- Gallery image 2: the xR&D workflow or system architecture.
- Gallery image 3: the output artifact.
- Gallery image 4: the feedback/operation loop.

The images should look like a family. Do not mix dark sci-fi, airy architectural render, and generic office photography on the same page.

### Insights

Goal: editorial intelligence.

Use lighter, quieter images: notebooks, diagrams, studio details, coastal context, annotated artifacts, process fragments, or abstract visual essays. These should feel like field notes from a working AI-native studio.

Avoid making every insight look like a product launch.

### Research

Goal: practical R&D, not academic cosplay.

Use images that feel like working notes, experiments, evaluation harnesses, interface tests, diagrams, and prototype artifacts. Keep the visual system precise, clean, and calm.

Avoid fake scientific charts, unreadable equations, or exaggerated lab imagery.

### Careers

Goal: show the work culture and talent pipeline.

Use human-centered studio and training scenes: mentorship, paired work, critique sessions, project reviews, classroom-to-studio movement. The mood should be ambitious, warm, and hands-on.

Avoid "startup party" imagery or inflated corporate recruiting visuals.

### Contact

Goal: make the company feel reachable and grounded.

Use location and studio images: Da Nang daylight, entrance or workspace details, calm meeting areas, map-like line motifs, or coastal urban context. The image should lower friction, not perform grandeur.

## 11. Prompt Template

Use this structure for every generated asset:

```text
Use case: website brand image
Asset type: <page + slot, e.g. Project detail hero>
Brand context: xR&D Technologies is an AI-native company from Da Nang, Vietnam. It builds with AI, creates with AI, supports AI adoption, and grows AI-native talent.
Page context: <what this page/section says>
Primary subject: <one clear subject>
Scene/backdrop: <Da Nang studio, coastal city, product mockup, workshop, abstract system, etc.>
Style/medium: premium editorial photorealism with restrained generated-interface overlays
Composition: <wide hero / card crop / square tile / device mockup; include negative space if needed>
Lighting: airy coastal daylight or deep navy evening only when the page needs contrast
Color palette: Ocean Blue #0B1D3A, Luminous Blue #1E68FF, AI Cyan #00D6FF, Concrete Gray #D9DBE1, Coastal Sand #F2E6D3; violet only as a rare accent
AI layer: thin line overlays, network nodes, half-wireframe traces, sparse grid fragments; integrated into the physical scene
People: <none / non-identifiable silhouettes / hands / small team / training scene>
Constraints: no readable text, no real logos, no client names, no watermarks, no robots, no cyberpunk, no overloaded command-center dashboards, no stock-photo posing
```

## 12. Negative Prompt Baseline

Apply this negative direction unless a page explicitly requires otherwise:

```text
Avoid robots, androids, glowing brains, server rooms as the main subject, crypto-style neon, cyberpunk cityscapes, dense holographic dashboard walls, unreadable fake text, fake brand logos, client names, exaggerated lens flare, decorative orbs, purple-heavy gradients, generic boardroom handshakes, headset call centers, stock-photo teams smiling at laptops, and overly dark blue command centers.
```

## 13. Quality Checklist

Before using an image in the site, check:

- Does it feel like xR&D, not a generic AI startup?
- Is the AI layer restrained and integrated?
- Is there a clear main subject?
- Does the image support the page content?
- Does it avoid invented client facts?
- Does it leave enough room for text where needed?
- Does it share the same palette and atmosphere as the reference brand images?
- Does it avoid over-dark, over-neon, over-dashboard aesthetics?
- Would it still make sense if all HTML text were removed?

## 14. Specific Correction From The First Test

The first generated `project-001` hero moved too far toward a dark AI command center. It matched the navy/blue palette, but it over-indexed on dashboards and operational screens.

For future project imagery:

- Reduce interface density.
- Use more coastal daylight, glass, concrete, and human workspace cues.
- Show one clear workflow or artifact, not an entire control room.
- Let AI appear as a precise structural overlay, not a spectacle.
- Keep the image premium and quiet enough to sit under large editorial typography.
